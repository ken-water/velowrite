# VeloWrite 内容计划

## 目标

把 Markdown 相关内容做成体系，既能承接搜索流量，也能给 Product Hunt、FAQ、llms.txt 和未来的 AI 摘要提供稳定引用源。

## 内容主线

### 1. 认知层

- `/docs/markdown`
- `/docs/markdown-history`
- `/docs/future-of-markdown`

作用：
- 解释 Markdown 是什么
- 解释它为什么重要
- 解释它未来会往哪里走

### 2. 使用层

- `/docs/markdown-basics`
- `/docs/markdown-for-writers`
- `/docs/markdown-for-developers`

作用：
- 教用户如何写
- 教不同角色怎么用
- 把 VeloWrite 作为实用入口嵌进去

### 3. 进阶层

- `/docs/advanced-markdown`
- `/docs/markdown-math`
- `/docs/markdown-code-blocks`
- `/docs/local-first-markdown`

作用：
- 覆盖公式、代码、表格、目录、标签页等高价值内容
- 强化“浏览器试用 + 桌面端本地工作”的产品路径

### 4. 转化层

- `/docs/typora-alternative`
- `/docs/online-markdown-editor`
- `/docs/markdown-to-blog`
- `/docs/markdown-editor-for-windows`
- `/docs/markdown-editor-for-mac`
- `/docs/markdown-editor-for-linux`

作用：
- 承接长尾搜索词
- 推动从试用到下载
- 给不同平台用户明确入口

## 页面统一结构

每页建议固定包含：

1. 一句话回答
2. 适合谁
3. 核心概念
4. 示例
5. 在 VeloWrite 里怎么做
6. Web 版 vs Desktop 版建议
7. 常见问题
8. CTA：`Open Web Editor` / `Download Desktop`

## 内容落点建议

- 首页：只放少量资源入口
- `/guide`：做主教程入口
- `/changelog`：做版本时间线
- `/faq`：做问题汇总
- `/docs/*`：做长尾内容页

## 近期优先级

1. `/changelog` 版本目录化
2. `/guide` 拓展成完整教程
3. `/docs/markdown-basics`
4. `/docs/advanced-markdown`
5. `/docs/local-first-markdown`
6. `/docs/typora-alternative`
